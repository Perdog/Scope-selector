If you're like me, you miss the ability that users had with the XML API page that allowed them to select which scopes third-party apps could use.

Well as it turns out, it's still possible, with a little extra work.

Thankfully, I've done that work for you. When requesting scopes for your app, you are not required to request all scopes that it can use. While you aren't allowed to request scopes that aren't selected on your developer page, you can leave scopes out.

This page was designed to help with this process.

Simply replace the begining part of your URL redirect https://login.eveonline.com/oauth/authorize with https://eve-scopes.com, leave all of your additional search parameters in place.

When a user is redirected here, and selects the scopes they want to provide access to, this page will then rebuild the URL containing only those scopes. All information that you passed to this page will then be reused, including the redirect_uri that points back to your page, so absolutely no extra work is required from you (Unless you count putting in checks to make sure you have permission to use endpoints before calling them as work).


Example:

https://login.eveonline.com/oauth/authorize?response_type=[type]&redirect_uri=[redirect]&client_id=[id]&scope=esi-corporations.read_corporation_membership.v1 esi-characters.read_corporation_roles.v1 esi-corporations.track_members.v1 esi-corporations.read_titles.v1 esi-corporations.read_fw_stats.v1&state=[unique state]

becomes

https://eve-scopes.com/?response_type=[type]&redirect_uri=[redirect]&client_id=[id]&reqscope=esi-corporations.read_corporation_membership.v1&scope=esi-characters.read_corporation_roles.v1 esi-corporations.track_members.v1 esi-corporations.read_titles.v1 esi-corporations.read_fw_stats.v1&state=[unique state]

You can even click the above URL example to load what the page will look like. It won't be functional, because the parameters don't have proper values set, but you'll be able to see what the selection page looks like.

Note that the example link above also has an optional parameter you can provide called "reqscope". Any scopes provided under that parameter will be checked and disbaled on the selection page, allowing you to have core features alongside optional features.
